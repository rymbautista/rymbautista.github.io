Final Project: Froggy Friend

Project Description

Here is the latest version of my final Project "Froggy Friend!" In this version, you control a fly that will aid you in taking care of your froggy friend. You will balance bringing over items to entertain and feed your pal as their happiness and hunger levels decrease over time. Rename your friend and listen to music as you now have a new companion with you!

Play my game here!



Development Process

Adding Background and Images

The first thing I did after making my prototype was add in the background images and colors to make the room not blank. After getting in the background I added the custom elements of the door, the shelf, the hunger icon, and the happy icon. The static image was the fly cursor, I was having issues with having the fly being an animation so a static fly image was used.

Sprite interactions

After implementing the images I began adding the other animations to indicate that the frog has received the given item. This process utilized the addani function and called another function that passed in the item type to set what animation to change it to. There is also another if statement in the draw loop that changes the animation back to the default animation.

Painting

After implementing the different animations I worked on the painting sprite, which originally was going to use the imagur API but now uses Lorem Picsum to generate a random stock image to serve as the froggy friend's paintings.

Buttons

Next, I added the buttons. The first button that I implemented was for renaming your frog which makes an alert that prompts the user to input a new name. This new name is stored using p5's store item function which stores the name in the browser so that users can return to the pet that they named. Next came the music button, which loads in music when pressed and then stops when pressed again. The last button that I implemented was the help button which gives the user a brief description of how to take care of their froggy friend.

Status Boxes

The Final thing I implemented was the status boxes to indicate how your froggy friend is feeling when they get to certain hungry or happy levels. This was done using if statements that check what values the happy and hunger variables were at. This also included the background boxes that pop up with the messages which change colors depending on how low that level is.

Final Touches

Lastly were little touch-ups, such as adding a border around the painting to frame it, and repositioning the items on the shelves to be dependent on the screen width in addition to being more evenly and consistently placed.

Issue Deep-Dive

Changed Animations Not Staying

The biggest issue that I encountered was with changing animations at first I was having problems with the animations staying but it was an issue of calling the default animation in the draw loop instead of the setup draw frog function that I had. The last issue was that the animations weren't going to the right ones at first but it was an issue of not having enough = signs.

Music Not Playing

When dealing with music I ran into an issue where the music couldn't auto-play as a feature of my browser, how I made it so that the user has to press the button at least once to be able to listen to music and then press it again to turn it off.
