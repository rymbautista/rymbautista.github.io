Overview and usage

This is a game called Frapple that utilizes p5 and p5.play to create a game about a frog that catches apples falling from the sky.

Play Frapple!


Development Process

Ideation

The development process started actually with a build of frogger that I found online. Although this had a great foundation for a game, I found that I was built weirdly that it was very hard coded to the dimensions that it originally had, so I pivoted to a game where apples fall from the sky and the player controls the frog with their mouse to catch these apples.

The Sprites

The first thing I played with was creating sprites. I made the frog sprite first having it only move with the mouse and only left and right. Then I started to play with the falling objects. After getting one fall from the middle of the screen I was able to get them to randomly spawn from one of the set spawn points with different velocities.

Having them interact

After constructing the sprites I used overlap to check if they would intersect and if it did the player's score would go up, and then had it so that if the objects fell lower than the floor the player would lose a life. If they lost 3 the game was over.

Restart

The last function that I added was making the 'R' key on the keyboard restart the game for a game over if the user wanted to just restart. This led to me using an if statement for checking if the 'R' key has been pressed and then resetting the values to their default and clearing the array of apple objects that were currently falling.

Making it look Pretty

One of the last steps after implementing all the functional components of my game was making the game look more pretty. I created images of frogs and apples for the catcher and object sprites. I then started by adding background elements like the floor and a bar at the top, then I added an image of the hills in the back to fill the sky. Lastly, I added a box around the game over screen for readability.

Issues

Deploying to git hub pages

The biggest roadblock I faced was getting over the hurdle of deploying to git hub pages. This was mainly to small things in my code and missing files when doing the rollup. I was able to successfully deploy it after being pointed to some discord posts and some spot-checking of my code from Hannah.

Getting the hang of p5.play

One of the things I did struggle a tiny bit with was just learning how p5.play works and getting it to interact with p5 it just took me a bit of time to see how it can interact. I found that the references on the website were really helpful for learning about how these libraries work.
